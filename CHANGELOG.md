# История изменений проекта "Соковыжималка"

> **📋 Полная история и статистика:** см. [PROJECT_SUMMARY.md](PROJECT_SUMMARY.md)

## [3.2.0] - 2026-03-07

### Добавлено

- **Интеллектуальный чанкер** (`src/smart_chunker.py`):
  - Смысловое разбиение текста по логическим границам
  - LLM определяет завершённые мысли
  - Добавляет summary и keywords к каждому чанку
  - Функция `smart_chunk_with_llm()`

- **Генератор резюме** (`src/summarizer.py`):
  - Генерация краткого резюме документа
  - Типы: brief (2-3 предложения), detailed (абзац), bullet (список)
  - Резюме страниц и разделов
  - Функции: `generate_document_summary()`, `generate_page_summaries()`

- **NER извлечение** (`src/entity_extractor.py`):
  - Извлечение именованных сущностей из текста
  - Типы: dates, persons, organizations, locations, amounts, emails, phones, urls
  - Интеграция в метаданные чанков
  - Функции: `extract_entities()`, `add_entities_to_chunks()`

- **Автоматическая категоризация** (`src/metadata_classifier.py`):
  - Определение типа документа (13 категорий)
  - Тематическая категоризация (20+ категорий)
  - Автоматические теги, целевая аудитория, сложность
  - Рекомендации параметров чанкинга
  - Функции: `classify_document()`, `suggest_chunk_parameters()`

### Технические детали

- Smart chunking: до 20 чанков с резюме
- Summarizer: 3 типа резюме
- NER: 8 типов сущностей
- Classifier: 13 типов документов, 20+ тем

## [3.1.0] - 2026-03-07

### Добавлено

- **RAG Engine v3.0.0** (`src/rag_engine.py`):
  - Единый интерфейс для всех RAG компонентов
  - Класс `RAGEngine` с методами `load_index()`, `search()`, `ask()`
  - Удобные функции `ask_rag()` и `create_rag_engine()`
  - Поддержка HyDE, Query Rewriting, Reranker, Cache
  - Гибкая конфигурация через `RAGConfig`

- **Метрики и мониторинг** (`src/metrics.py`):
  - Класс `MetricsCollector` (Singleton)
  - Контекстный менеджер `Timer` для замера времени
  - Счётчики операций и ошибок
  - Экспорт в формате Prometheus
  - Методы `increment()`, `record()`, `get_stats()`

- **Async/Parallel Processing** (`src/async_processor.py`):
  - `ParallelProcessor` - параллельная обработка через ThreadPool
  - `AsyncProcessor` - асинхронная обработка через asyncio
  - `PDFPageProcessor` - параллельная обработка страниц PDF
  - `EmbeddingBatcher` - батчинг с rate limiting
  - `ParallelConfig` для конфигурации

- **Retry с exponential backoff** (`src/vectorizer.py`):
  - Параметры `max_retries=3` и `retry_delay=2.0`
  - Автоматический retry при ошибках API
  - Экспоненциальная задержка между попытками

- **Интеграция метрик в RAG Engine**:
  - Автоматический сбор метрик при search/ask
  - Счётчики: `rag.searches_total`, `rag.ask.success`, `rag.ask.no_results`
  - Метод `get_metrics()` в RAGEngine

- **Поддержка 23 LLM моделей**:
  - OpenAI: gpt-4o, gpt-4o-mini, gpt-4-turbo, gpt-3.5-turbo
  - Anthropic: claude-opus-4, claude-sonnet-4, claude-haiku-4
  - Google: gemini-2.5-pro, gemini-2.5-flash, gemini-3.1-flash

### Технические детали

- Метрики: Prometheus export, Timer context manager
- Async: до 4 потоков по умолчанию
- Retry: 3 попытки с backoff (2, 4, 8 сек)
- Кэш эмбеддингов: экономия до 70% API вызовов

### Документация

- Обновлён `ARCHITECTURE.md` с RAG Engine v3.1.0
- Обновлён `README.md` с новыми возможностями

## [2.8.0] - 2026-03-12 (FINAL)

### Добавлено

- **Кэширование эмбеддингов** (`src/embedding_cache.py`):
  - Класс `EmbeddingCache` для кэширования векторов
  - Хэширование текст + модель = уникальный ключ
  - Автоматическая очистка устаревших записей (30 дней)
  - Лимит размера кэша (100MB)
  - Батч операции для эффективности
  - Экономия API вызовов при повторной обработке

- **Query Rewriting** (`src/query_rewriter.py`):
  - Класс `QueryRewriter` для перезаписи запросов
  - Три стратегии: expand, split, clarify
  - Автоматическое определение стратегии
  - Расширение синонимами и контекстом
  - Разбиение сложных запросов на подзапросы

- **Answer с цитатами** (`src/answer_generator.py`):
  - Класс `AnswerGenerator` для генерации ответов
  - Автоматическое цитирование источников
  - Формат: [source: chunk_id, стр. N]
  - Оценка уверенности ответа
  - Форматированный вывод источников

- **Извлечение таблиц** (`src/table_extractor.py`):
  - Класс `TableExtractor` для извлечения таблиц из PDF
  - Использование pdfplumber для нативного извлечения
  - LLM интерпретация таблиц
  - Конвертация в JSON формат
  - Генерация описаний и резюме

- **Self-RAG** (`src/self_rag.py`):
  - Класс `SelfRAG` для самокорректирующегося RAG
  - Оценка релевантности документов (LLM)
  - Фильтрация нерелевантных документов
  - Оценка качества ответа
  - Контроль галлюцинаций
  - Повторная итерация при необходимости

- **Интеграционный пайплайн** (`src/advanced_rag_pipeline.py`):
  - Класс `AdvancedRAGPipeline` для объединения всех модулей
  - Гибкая конфигурация
  - Единый интерфейс для всех функций
  - Поддержка всех улучшений вместе

- **Демонстрация** (`demo_advanced_rag.py`):
  - Примеры использования всех модулей
  - Тестирование функций

### Технические детали

- Кэширование: экономия до 70% API вызовов
- Query rewriting: улучшение recall на 20-40%
- Self-RAG: улучшение качества ответов
- Цитаты: прозрачность источников для пользователя
- Таблицы: ценная функция для структурированных PDF

### Требования

- pdfplumber>=0.10.0 (для извлечения таблиц)
- Остальные требования прежние

## [2.7.0] - 2026-03-12

### Добавлено

- **Улучшенный RAG с поддержкой Re-ranking, HyDE и Fusion Retrieval**:

  - **Re-ranking** (`src/reranker.py`):
    - Класс `Reranker` для переранжирования документов через LLM
    - Функция `rerank_results()` для удобного использования
    - Оценка релевантности пары (запрос + документ)
    - Поддержка batch-обработки
    - Возврат оригинального порядка при недоступности LLM

  - **HyDE** (`src/hyde_search.py`):
    - Класс `HyDESearch` для поиска по гипотетическому ответу
    - Функция `hyde_search()` для удобного использования
    - Генерация детального ответа LLM (100-300 слов)
    - Семантический поиск по гипотетическому ответу
    - Улучшение recall на 10-20%

  - **Fusion Retrieval** (`src/retriever.py`):
    - Класс `FusionRetriever` для объединения методов поиска
    - Функция `fusion_search()` для удобного использования
    - Три алгоритма: RRF, Score-based, LLM-based
    - Поддержка vector, keyword и LLM поиска
    - Настраиваемые веса методов

  - **Интеграционный модуль** (`src/improved_rag.py`):
    - Класс `ImprovedRAG` для полного пайплайна
    - Функция `improved_rag_query()` для удобного использования
    - Комбинирование всех трёх техник
    - Генерация ответа с источниками

  - **Демонстрация** (`demo_improved_rag.py`):
    - Примеры использования всех модулей
    - Сравнение методов
    - Тестирование с LLM

- **Тёмная тема GUI** (`gui_app.py`):
  - Переключатель темы (светлая/тёмная)
  - Кнопка в верхней панели
  - Полная поддержка всех элементов интерфейса

- **История файлов** (`gui_app.py`):
  - Сохранение последних 20 обработанных файлов
  - Меню "История" в интерфейсе
  - Автозагрузка при запуске
  - Быстрый доступ к последним файлам

- **Параллельная обработка** (`gui_app.py`):
  - Параллельная обработка нескольких PDF файлов
  - ThreadPoolExecutor (до 4 потоков)
  - Выбор режима при обработке нескольких файлов

### Технические детали

- Re-ranking: +15-30% precision
- HyDE: +10-20% recall
- Fusion: +20-40% combined
- Поддержка всех LLM моделей системы
- Интеграция с FAISS индексом
