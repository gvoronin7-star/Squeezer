## Обзор безопасности

Проект "Соковыжималка" (Squeezer) следует современным практикам безопасности для защиты данных и систем. Этот документ описывает меры безопасности и рекомендации для безопасного использования системы.

---

## Важные уведомления о безопасности

### API ключи

- **Никогда не коммитите** `.env` файл с API ключами в репозиторий
- **Никогда не делитесь** API ключами публично
- **Используйте переменные окружения** для хранения чувствительных данных
- **Регулярно обновляйте** API ключи

### Данные

- **PDF файлы могут содержать конфиденциальные данные** — обрабатывайте их с осторожностью
- **Векторные базы данных** содержат эмбеддинги ваших документов — защищайте их
- **Логи** могут содержать чувствительные данные — очищайте их перед распространением

---

## Меры безопасности

### 1. Защита API ключей

✅ **Рекомендуется:**

- Используйте файл `.env` для хранения API ключей
- Добавьте `.env` в `.gitignore`
- Используйте разные API ключи для разных окружений (dev, staging, prod)
- Регулярно обновляйте API ключи
- Используйте прокси для скрытия реальных API ключей

❌ **Не рекомендуется:**

- Хардкодить API ключи в коде
- Коммитить `.env` файл в репозиторий
- Делиться API ключами в чатах или публичных местах
- Использовать один API ключ для всех проектов

### 2. Защита данных

✅ **Рекомендуется:**

- Сканируйте PDF файлы на наличие вирусов перед обработкой
- Ограничивайте доступ к PDF файлам и векторным базам данных
- Используйте шифрование для хранения чувствительных данных
- Создавайте бэкапы и защищайте их
- Следуйте принципу минимальных привилегий

❌ **Не рекомендуется:**

- Обрабатывать файлы из ненадёжных источников без проверки
- Хранить чувствительные данные в открытом виде
- Делиться векторными базами данных публично без необходимости

### 3. Безопасность окружения

✅ **Рекомендуется:**

- Используйте виртуальное окружение Python (`venv`)
- Обновляйте зависимости регулярно
- Используйте актуальные версии Python и библиотек
- Ограничивайте сетевой доступ
- Используйте файрвол

❌ **Не рекомендуется:**

- Запускать от имени root/administrator без необходимости
- Использовать устаревшие версии зависимостей
- Отключать антивирусное ПО

### 4. Безопасность логов

✅ **Рекомендуется:**

- Регулярно очищайте логи
- Не логируйте чувствительные данные (API ключи, пароли)
- Ограничивайте доступ к файлам логов
- Используйте ротацию логов

❌ **Не рекомендуется:**

- Логировать чувствительные данные
- Хранить логи indefinitely без очистки
- Делиться логами публично без очистки

## Уязвимости

### Известные уязвимости

На текущий момент (версия 2.4.0) известных критических уязвимостей нет.

### Потенциальные риски

1. **Инъекция кода через PDF файлы**
   - **Риск:** Низкий
   - **Митигация:** Сканируйте PDF файлы на вирусы, используйте библиотеки для безопасной обработки

2. **Утечка API ключей**
   - **Риск:** Средний
   - **Митигация:** Используйте переменные окружения, не коммитите `.env`

3. **DoS атаки через большие файлы**
   - **Риск:** Низкий
   - **Митигация:** Ограничьте размер файлов (максимум 50 МБ)

4. **SQL инъекция (при использовании БД)**
   - **Риск:** Низкий
   - **Митигация:** Используйте параметризованные запросы

## Безопасность в Production

### Checklist перед развертыванием в production

- [ ] Убедитесь, что `.env` файл не коммитится
- [ ] Используйте отдельные API ключи для production
- [ ] Настройте правильные права доступа к файлам
- [ ] Включите логирование и ротацию логов
- [ ] Настройте бэкапы
- [ ] Ограничьте сетевой доступ
- [ ] Используйте HTTPS (если есть веб-интерфейс)
- [ ] Обновите все зависимости до последних версий
- [ ] Протестируйте систему в staging окружении
- [ ] Настройте мониторинг и оповещения

### Мониторинг

- Регулярно проверяйте логи на наличие подозрительной активности
- Мониторьте использование API ключей (количество запросов)
- Следите за использованием ресурсов (CPU, RAM, диск)
- Настройте оповещения о критических ошибках
