# Архитектура проекта "Соковыжималка"

**Версия:** 3.3.0
**Автор:** [Line_GV](https://t.me/Line_GV)

## Обзор

Проект "Соковыжималка" представляет собой систему для обработки PDF-документов и подготовки данных для использования в RAG-системах. Система состоит из пяти основных модулей, каждый из которых выполняет определённый этап обработки, а также включает систему бэкапа и восстановления.

## Модули

### Модуль 1: Каркас и конфигурация

**Файлы:**
- `squeezer.py` - CLI приложение
- `gui_app.py` - GUI приложение
- `config.json` - Конфигурация
- `.env` - Переменные окружения
- `.env.example` - Шаблон переменных окружения
- `VERSION` - Версия системы

**Функциональность:**
- Обработка аргументов командной строки
- Графический интерфейс на Tkinter
- Загрузка конфигурации
- Загрузка переменных окружения
- Настройка логирования

### Модуль 2: Предобработка текста

**Файлы:**
- `src/preprocessor.py` - Основной модуль предобработки

**Функции:**
- `extract_text_from_pdf()` - Извлечение текста из PDF
- `clean_text()` - Очистка данных
- `normalize_text()` - Нормализация данных
- `structure_text()` - Структурирование текста
- `process_pdf()` - Полный цикл обработки (этапы 1-8)

### Модуль 2.1: Пользовательский интерфейс

**Файлы:**
- `src/ui/pdf_loader.py` - UI для загрузки PDF

**Классы:**
- `PDFLoader` - Загрузка и валидация PDF
- `PDFLoaderUI` - Графический интерфейс
- `SqueezerApp` - Основное приложение

### Модуль 3: Чанкинг и метаданные

**Файлы:**
- `src/chunker.py` - Модуль чанкинга

**Функции:**
- `hybrid_chunking()` - Гибридное разбиение текста
- `add_metadata()` - Добавление метаданных
- `validate_chunks()` - Валидация чанков
- `generate_chunking_report()` - Генерация отчёта
- `generate_chunking_demo()` - Генерация демонстрации
- `process_chunks()` - Полный цикл обработки чанков

### Модуль 4: Векторизация и векторная БД

**Файлы:**
- `src/vectorizer.py` - Модуль векторизации

**Функции:**
- `validate_data()` - Проверка качества данных (этап 7)
- `save_to_vector_db()` - Генерация эмбеддингов и создание индекса (этап 8)
- `generate_vectorization_report()` - Генерация отчёта
- `process_vectorization()` - Полный цикл обработки (этапы 7-8)

**Зависимости:**
- `openai` - OpenAI API для эмбеддингов
- `faiss-cpu` - FAISS для векторной БД
- `numpy` - Работа с массивами
- `python-dotenv` - Загрузка переменных окружения

### Модуль 5: RAG Builder

**Файлы:**
- `rag_builder.py` - Основное приложение RAG Builder
- `src/rag_instructions.py` - Генерация инструкций и примеров
- `src/ui/rag_builder_ui.py` - GUI интерфейс

**Функции:**
- `generate_rag_package()` - Создание полного пакета RAG-базы
- `generate_rag_readme()` - Генерация README для RAG-базы
- `generate_connection_example_python()` - Генерация примера подключения
- `create_rag_base()` - Создание RAG-базы из PDF-файлов

**Классы:**
- `RAGBuilder` - Основной класс приложения
- `RAGBuilderUI` - GUI интерфейс
- `RAGFileLoader` - Загрузка и валидация PDF-файлов

**Функциональность:**
- Пакетная обработка нескольких PDF-файлов
- Автоматическая генерация названия RAG-базы
- Создание готового пакета с инструкциями
- Копирование исходных PDF-файлов в базу
- Генерация примеров кода для подключения

### Система бэкапа и восстановления

**Файлы:**
- `create_backup.py` - Скрипт создания бэкапа
- `restore_backup.py` - Скрипт восстановления
- `BACKUP_GUIDE.md` - Руководство по бэкапу

**Функции:**
- `create_backup()` - Создание бэкапа системы
- `list_backups()` - Список доступных бэкапов
- `restore_from_backup()` - Восстановление из бэкапа
- `should_exclude()` - Проверка исключаемых файлов

**Функциональность:**
- Создание бэкапов с датой и временем
- Исключение временных файлов и выходных данных
- Выбор бэкапа из списка
- Режим проверки (dry-run) перед восстановлением

## Поток данных

### Поток обработки (CLI)

```
PDF файл → squeezer.py → process_pdf()
                                    ↓
                            extract_text_from_pdf()
                                    ↓
                                clean_text()
                                    ↓
                              normalize_text()
                                    ↓
                              structure_text()
                                    ↓
                            hybrid_chunking() [опционально]
                                    ↓
                              add_metadata() [опционально]
                                    ↓
                            validate_data() [опционально]
                                    ↓
                            save_to_vector_db() [опционально]
                                    ↓
                            generate_report()
                                    ↓
                          generate_chunking_report() [опционально]
                                    ↓
                          generate_chunking_demo() [опционально]
                                    ↓
                      generate_vectorization_report() [опционально]
```

### Поток обработки (GUI)

```
PDF файл → PDFLoaderUI → SqueezerApp
                                   ↓
                            process_pdf()
                                   ↓
                             (как в CLI)
```

```
Чанки → validate_data()

        Проверка качества данных

                ↓
        Создание индекса FAISS
                ↓
        Сохранение в output/vector_db/
                ↓
```

```
            ↓
```

## Валидация

### Валидация PDF файлов
- Проверка формата (.pdf)
- Проверка размера (максимум 50 МБ)
- Проверка целостности файла

### Валидация чанков (Модуль 3)
- Проверка на пустоту
- Проверка на слишком короткие чанки (< 10 символов)
- Статистика по чанкам

### Валидация данных перед векторизацией (Модуль 4)
- Подсчёт пустых чанков
- Вычисление средней длины чанков
- Статистика по данным (мин./макс. длина)

## Расширяемость

### Добавление новых этапов обработки

1. Создайте функцию в `src/preprocessor.py` или новом модуле
2. Добавьте вызов функции в `process_pdf()`
3. Обновите документацию

### Добавление новых форматов входных данных

1. Создайте функцию извлечения текста для нового формата
2. Интегрируйте её в `extract_text_from_pdf()`
3. Обновите документацию

### Добавление новых моделей эмбеддингов

1. Укажите название модели в `config.json` или при вызове функции
2. Модуль автоматически использует указанную модель через OpenAI API
3. Обновите документацию
