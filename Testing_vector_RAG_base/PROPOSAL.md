**Версия:** 1.0  
**Дата:** 2026-03-23  
**Автор:** Line_GV

---

## 1. Обзор

### 1.1 Цель

Создать комплексную систему для оценки качества RAG-базы данных (векторной БД + датасет чанков).

### 1.2 Задачи

1. **Валидация структуры** — проверка целостности файлов
2. **Оценка качества чанков** — размер, содержание, метаданные
3. **Тестирование поиска** — релевантность, полнота
4. **Оценка качества ответов** — точность, groundedness
5. **Проверка покрытия** — достаточно ли информации
6. **Генерация отчётов** — визуализация результатов

## 3. Компоненты

### 3.1 RAG Quality Tester (главный модуль)

**Файл:** `rag_quality_tester.py`

**Класс:** `RAGQualityTester`

**Методы:**
- `load_database(db_path)` — загрузка RAG-базы
- `run_all_tests()` — запуск всех тестов
- `run_category(category)` — запуск тестов категории
- `get_score()` — получение общего балла качества
- `generate_report()` — генерация отчёта

**Категории тестов:**
| Категория | Вес | Описание |
|-----------|-----|----------|
| Structure | 10% | Целостность файлов |
| Chunks | 25% | Качество чанков |
| Search | 30% | Качество поиска |
| Answers | 25% | Качество ответов |
| Coverage | 10% | Покрытие контента |

### 3.2 Тесты качества чанков

**Файл:** `test_chunk_quality.py`

**Метрики:**

| Метрика | Формула | Порог |
|---------|---------|-------|
| Размер чанка | `len(chunk.text)` | 100-1000 символов |
| Пустые чанки | `count(empty)` | 0 |
| Дубликаты | `similarity > 0.95` | < 5% |
| Полнота метаданных | `required_fields` | 100% |
| Когерентность | LLM оценка | > 0.7 |

**Тесты:**
```python
class ChunkQualityTests:
    def test_chunk_sizes()        # Распределение размеров
    def test_empty_chunks()       # Пустые чанки
    def test_duplicates()         # Дубликаты
    def test_metadata()           # Метаданные
    def test_coherence()          # Когерентность (LLM)
    def test_overlap_quality()    # Качество перекрытия
```

### 3.3 Тесты качества поиска

**Файл:** `test_search_quality.py`

**Метрики:**

| Метрика | Описание | Формула |
|---------|----------|---------|
| Precision@K | Точность топ-K | relevant / K |
| Recall@K | Полнота топ-K | found / total_relevant |
| MRR | Mean Reciprocal Rank | avg(1/rank) |
| NDCG | Normalized DCG | оценка ранжирования |
| Latency | Время поиска | среднее время |

**Тесты:**
```python
class SearchQualityTests:
    def test_precision_at_k()     # Precision@5, @10
    def test_recall_at_k()        # Recall@5, @10
    def test_mrr()                # Mean Reciprocal Rank
    def test_latency()            # Время поиска
    def test_edge_queries()       # Граничные случаи
    def test_multilingual()       # Мультиязычность
```

## 4. Пример использования

### CLI

```bash
# Полное тестирование
python rag_quality_tester.py --db-path ./output/vector_db --report ./reports/

# Только качество чанков
python rag_quality_tester.py --db-path ./output/vector_db --category chunks

# Сравнение двух баз
python rag_quality_tester.py compare ./db1 ./db2
```

### Python API

```python
from Testing_vector_RAG_base.rag_quality_tester import RAGQualityTester

# Создаём тестировщик
tester = RAGQualityTester(
    db_path="./output/vector_db",
    api_key="sk-...",
    llm_model="gpt-4o-mini"
)

# Запускаем все тесты
results = tester.run_all_tests()

# Получаем общий балл
score = tester.get_score()
print(f"Общая оценка: {score}/100")

# Генерируем отчёт
tester.generate_report("./reports/quality_report.md")
```
