# Руководство по бэкапу и восстановлению

## Обзор

Система "Соковыжималка" включает инструменты для создания бэкапов и восстановления проекта.

## Инструменты

### 1. Создание бэкапа (`create_backup.py`)

Создаёт полную копию системы в папку `backups/`.

**Что копируется:**
- Все исходные файлы проекта (`src/`, `ui/`, и т.д.)
- Конфигурационные файлы
- Документация

**Что НЕ копируется:**
- Временные файлы (`__pycache__`, `*.pyc`)
- Логи (`logs/`, `*.log`)
- Виртуальное окружение (`venv/`, `.env`)
- Выходные данные (`output/`, `rag_bases/`)
- PDF файлы (`pdfs/`)
- Git метаданные (`.git/`)
- IDE настройки (`.vscode/`, `.idea/`)
- Скрипты бэкапа и папка бэкапов (`backups/`, `create_backup.py`, `restore_backup.py`)

**Использование:**
```bash
python create_backup.py
```

**Результат:**
- Создаётся папка `backups/squeezer_backup_YYYYMMDD_HHMMSS/`
- Выводится статистика бэкапа

---

### 2. Восстановление из бэкапа (`restore_backup.py`)

Восстанавливает систему из выбранного бэкапа.

**Использование:**
```bash
python restore_backup.py
```

**Действия:**
1. Показывает список доступных бэкапов
2. Позволяет выбрать нужный бэкап
3. Предлагает выбрать режим:
   - **Обычный** - копирует файлы в проект
   - **Проверка** - только показывает что будет сделано

**Важно:**
- Файлы, которые уже существуют в проекте, не перезаписываются
- Можно использовать режим проверки перед реальным восстановлением

## Рекомендации по бэкапу

### Перед обновлением версии
```bash
# 1. Создайте бэкап
python create_backup.py

# 2. Установите новую версию
pip install --upgrade squeezer-rag

# 3. Если что-то пошло не так, восстановитесь
python restore_backup.py
```

### Перед критическими изменениями
```bash
# Создайте бэкап перед изменением кода
python create_backup.py

# Внесите изменения в код
# ...

# Если нужно откатиться
python restore_backup.py
```

### Регулярное резервное копирование
Рекомендуется создавать бэкапы:
- Перед каждым крупным обновлением
- После завершения важной работы
- Перед изменением конфигурации

## Управление бэкапами

### Удаление старых бэкапов

**Windows (PowerShell):**
```powershell
# Удалить бэкапы старше 30 дней
Get-ChildItem "backups\squeezer_backup_*" | Where-Object {
    $_.LastWriteTime -lt (Get-Date).AddDays(-30)
} | Remove-Item -Recurse -Force

# Оставить только последние 5 бэкапов
Get-ChildItem "backups\squeezer_backup_*" |
    Sort-Object LastWriteTime -Descending |
    Select-Object -Skip 5 |
    Remove-Item -Recurse -Force
```

**Linux/Mac:**
```bash
# Удалить бэкапы старше 30 дней
find backups/ -name "squeezer_backup_*" -type d -mtime +30 -exec rm -rf {} +

# Оставить только последние 5 бэкапов
cd backups/
ls -t squeezer_backup_* | tail -n +6 | xargs rm -rf
```

## Автоматизация бэкапа

### Windows (Task Scheduler)

Создайте scheduled task для автоматического бэкапа:

```powershell
# Создаём скрипт auto_backup.ps1
$scriptPath = "C:\path\to\squeezer\create_backup.py"
python $scriptPath
```

Настройте Task Scheduler для запуска этого скрипта по расписанию.

## Устранение проблем

### Ошибка "Permission denied"
**Решение:** Запустите скрипт от имени администратора или закройте все программы, использующие файлы проекта.

### Ошибка "No space left on device"
**Решение:** Освободите место на диске или удалите старые бэкапы.

### Бэкап не содержит нужных файлов
**Решение:** Проверьте, что файлы не находятся в исключаемых папках (`output/`, `pdfs/`, и т.д.).

### Восстановление не копирует файлы
**Решение:** Файлы уже существуют в проекте. Сначала удалите их или переместите в другое место.
