# Вклад в проект "Соковыжималка" (Squeezer)

Спасибо за интерес к внесению вклада в проект "Соковыжималка"!

## Как внести вклад

### Сообщение о проблемах (Issues)

Если вы нашли ошибку или хотите предложить улучшение:

1. Проверьте, нет ли уже существующего issue с похожей проблемой
2. Создайте новый issue с чётким описанием:
   - **Заголовок**: Краткое описание проблемы или предложения
   - **Описание**: Подробное описание, воспроизводимые шаги (для багов)
   - **Окружение**: Версия Python, версия проекта, ОС
   - **Желаемое поведение**: Что вы ожидаете
   - **Фактическое поведение**: Что происходит на самом деле
   - **Логи**: Соответствующие логи из папки `logs/`

### Pull Requests

1. **Форкните репозиторий**
2. **Создайте ветку** для вашего изменения:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Внесите изменения**:
   - Следуйте стилю кода существующего проекта
   - Добавьте комментарии к сложной логике
   - Обновите документацию при необходимости
4. **Запустите тесты** (если есть):
   ```bash
   python -m pytest tests/
   ```
5. **Зафиксируйте изменения**:
   ```bash
   git commit -m "Add: краткое описание изменений"
   ```
6. **Отправьте изменения**:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Создайте Pull Request** с описанием ваших изменений

### Стиль кода

- Используйте PEP 8 для Python кода
- Максимальная длина строки: 100 символов
- Используйте 4 пробела для отступов
- Добавляйте docstrings ко всем функциям и классам

### Коммиты

Используйте следующие префиксы для сообщений коммитов:

- `Add:` - добавление новой функциональности
- `Fix:` - исправление ошибки
- `Update:` - обновление существующего кода или документации
- `Refactor:` - рефакторинг без изменения функциональности
- `Docs:` - изменения в документации
- `Test:` - добавление или обновление тестов

Примеры:
- `Add: Добавлена поддержка новых форматов PDF`
- `Fix: Исправлена ошибка при векторизации больших файлов`
- `Docs: Обновлено руководство пользователя`

### Документация

При изменении функциональности обновите:
- README.md (если затронуты основные функции)
- USER_GUIDE.md (если затронут пользовательский интерфейс)
- API документацию в папке `docs/`
- CHANGELOG.md (для релизов)

## Требования

### Python

- Python 3.8 или выше
- Все зависимости из `requirements.txt`

### Зависимости

Установите зависимости:
```bash
pip install -r requirements.txt
```

## Тестирование

### Запуск тестов

```bash
python -m pytest tests/
```

### Добавление тестов

Для новых функций добавляйте тесты в папку `tests/`.

## Локализация

Проект поддерживает два языка:
- Русский ('ru')
- Английский ('en')

При добавлении нового текста в UI обеспечьте поддержку обоих языков.

## Ветвление

- `main` - основная ветка для production версии
- `develop` - ветка для разработки новых функций
- `feature/*` - ветки для конкретных функций
- `bugfix/*` - ветки для исправления ошибок
- `hotfix/*` - ветки для срочных исправлений в production

## Релизы

Релизы выполняются по схеме версионирования Semantic Versioning (SemVer):

- `MAJOR.MINOR.PATCH`
- MAJOR: несовместимые изменения API
- MINOR: новая функциональность с обратной совместимостью
- PATCH: исправление ошибок с обратной совместимостью

Примеры:
- `2.4.0` → `2.5.0` - новая функциональность
- `2.4.0` → `2.4.1` - исправление ошибки
- `2.4.0` → `3.0.0` - несовместимые изменения
